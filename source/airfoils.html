nameString

Label identifying the airfoils

coordinatesObject

The airfoil coordinates are specified here in the sub-fields x and y. x and y must have the same length. x must be defined between 0, which corresponds to the leading edge, and 1, which corresponds to the trailing edge. Airfoil coordinates should be defined from trailing edge (x=1) towards the suction side (mostly positive y values), to leading edge (x=0, y=0), to the pressure side (mostly negative y), and conclude at the trailing edge pressure side (x=1). Flatback airfoils can be defined either open (x=1, y!=0) or closed (x=1, y=0).

relative_thicknessFloat

Float defined between 0 (plate) and 1 (cylinder) to specify the relative thickness of the airfoil. This generates a small redundancy (it could be determined from the field coordinates), but it simplifies the converters.

aerodynamic_centerFloat

Float defined between 0 (leading edge) and 1 (trailing edge) to specify the chordwise coordinate of the aerodynamic center used to define the polars.

polars : Object List of one or more sets of polars for the airfoil. Each entry is marked with a dash (-) mark. The field polars must include the sub-fields configuration, re, c_l, c_d, and c_m.

configurationString

String to describe and identify the set of polars

reFloat

Reynolds number of the polars

c_l: Object

Lift coefficient as a function of angle of attack

c_d: Object

Drag coefficient as a function of angle of attack

c_m: Object

Moment coefficient as a function of angle of attack

c_l, c_d, and c_m are defined via arrays grid and values, which must have the same length. grid contains the angles of attack in radians and must be specified between -pi (-3.141592653589793) and pi (+3.141592653589793), while values contains the coefficients.

The airfoils listed in this database are not all necessarily used in the blade. Only the ones called in airfoil_position' within :code:`outer_shape_bem will actually be loaded to model the blade.
